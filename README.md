## Подготовительные шаги

### Установите `uv`

**Windows**

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

**MacOS / Linux**

```sh
curl -LsSf https://astral.sh/uv/install.sh | sh
```

После установки перезапустите редактор.

### Синхронизируйте зависимости

Откройте директорию в `Visual Studio Code` либо `PyCharm`.

Выполните в терминале команду синхронизации

```sh
uv sync
```

### Активируйте окружение

После предыдущего шага виртуальное окружение должно было синхронизироваться автоматически. Если это так, то в терминале перед путем к директории должно быть высвечено `(kt1)`, например

```sh
(kt1) C:\Users\user\Documents\python-kt-1
```

Если это не так, активируйте виртуальное окружение вручную

**Windows**

```powershell
.venv\Scripts\activate
```

**MacOS / Linux**

```sh
source .venv/bin/activate
```

Для проверки работоспособности можете выполнить следующую команду

```sh
uv version
```

Если версия определится, всё настроено корректно.


## Процесс работы над заданиями

Рекомендуем использовать подход `TDD`, то есть отталкиваться при разработке от автотестов. Они написаны и покрывают все необходимые и достаточные случаи, а в качестве раннера тестов используется `pytest`.

При разработке в этом подходе мы регулярно запускаем пакет тестов к реализуемому заданию, постепенно делая тесты зелёными.

### Пример для задания "Аббревиатуры"

В качестве примера запустим тесты для задания "Аббревиатуры" следующей командой, где опция *-k* дает нам возможность задать фильтр для тестов и не запускать все тесты разом:

```sh
pytest -k abbreviate
```

Результатом должно быть 9 непройденных тестов при 76 деактивированных.

Внесём изменения в файл `src/kt1/abbreviate.py`, реализуя наивное решение: разбить на слова сплитом и склеить первые буквы каждого.

```python

def abbreviate(phrase: str) -> str:
    """
    документация
    """

    return ''.join(word[0] for word in phrase.split())
```

Перезапустим тесты:

```sh
pytest -k abbreviate
```

Результатом будет уже лишь четыре проваленных теста, при пяти пройденных и 76 деактивированных.

Теперь можно посмотреть на конкретные проблемы:
- маленькие буквы необходимо приводить к заглавным
- дефисы внутри слов являются разделителями наравне с пробелами
- дефисы вне слов являются мусорными и не должны учитываться
- ровно как нижние подчеркивания и иная пунктуация.

Решим первую из проблем: начнем приводить к верхнему регистру перед слиянием символов:

```python

def abbreviate(phrase: str) -> str:
    """
    документация
    """

    return ''.join(word[0].upper() for word in phrase.split())
```

Перезапустим тесты - увидим, что прошел ещё один. 

По той же схеме, меняя фильтр при запуске автотестов, можно работать и с иными задачами.

## Работа в системе контроля версий

Создайте ветку со своей фамилией

```sh
git switch -c фамилия-измени-меня
```

Если вы выполнили предыдущий шаг, у вас уже есть изменения, готовые к коммиту. Выполните его:

```sh
git add .
git commit -m "работа над аббревиатурами"
```

Если гит выдает ошибку, почитайте и погуглите её: могут быть не настроены имя и почта, может отсутствовать сам гит.

Если всё ок, залейте вашу ветку на гитхаб:

```
git push -u origin фамилия-измени-меня
```

Далее можете делать (или не делать) промежуточные коммиты.

**Главное, не забудьте закоммитить и запушить финальные изменения!**
